## 对话创作、场景解析

可能需要的注意事项：
1.对话要有重试自动机制、手动触发重试机制



- 场景 1：
输入创作主题，可以 @ 书籍 or 世界观、大纲，等

比如 @书籍 1，发送 `开始世界观创作`

决策 Agent 介入，当前开始世界观创作了，可以干活了，是否有疑问的点，或者询问用户是否有需要补充的信息,开始 @3 个写作 Agent 开始写作

3 位写作 Agent 理解需求后，自主决定是否需要列出一些是否需要一些额外的用户补充或者存在待疑的东西，用户补充

补充完成后开始写作。

等待结束
打分员工对不同写作员工内容进行打分
留下综合评定最高的内容重新进入一个对话（减少上下文）
纠错员工开始纠错或提建议
最终胜出的写作员工，基于纠错或建议继续修改内容  diff
基于纠错后的内容基于打分，打分不通过继续修改
直到打分通过
工具调用对应的 世界观存储 API 进行存储
